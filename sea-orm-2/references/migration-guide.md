## Breaking Changes

### Entity Definition

**1.0 Pattern (Deprecated)**

```rust
#[derive(Clone, Debug, PartialEq, DeriveEntityModel)]
#[sea_orm(table_name = "user")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    pub name: String,
}

#[derive(Copy, Clone, Debug, EnumIter, DeriveRelation)]
pub enum Relation {
    #[sea_orm(has_many = "super::post::Entity")]
    Posts,
}

impl Related<super::post::Entity> for Entity {
    fn to() -> RelationDef {
        Relation::Posts.def()
    }
}
```

**2.0 Pattern (Preferred)**

```rust
#[sea_orm::model]
#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel)]
#[sea_orm(table_name = "user")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    pub name: String,
    #[sea_orm(has_many)]
    pub posts: HasMany<super::post::Entity>,
}

// Relation enum and Related trait impl no longer needed!
```

### API Renames

| 1.0                                              | 2.0                                                  |
| ------------------------------------------------ | ---------------------------------------------------- |
| `.into_condition()`                              | `.into()`                                            |
| `db.execute(Statement::from_sql_and_values(..))` | `db.execute_raw(Statement::from_sql_and_values(..))` |
| `db.query_all(backend.build(&query))`            | `db.query_all(&query)`                               |
| `Alias::new("col")`                              | `Expr::col("col")`                                   |
| `insert_many(..).on_empty_do_nothing()`          | `insert_many([])`                                    |

## New Entity Format

### The `#[sea_orm::model]` Macro

SeaORM 2.0 introduces a new `#[sea_orm::model]` macro that:

1. Generates `Model`, `ActiveModel`, and related types
2. Handles relations directly in the struct
3. Eliminates boilerplate `Relation` enum and `Related` trait implementations

### Relations in Model

```rust
// Has-One
#[sea_orm(has_one)]
pub profile: HasOne<super::profile::Entity>,

// Has-Many
#[sea_orm(has_many)]
pub posts: HasMany<super::post::Entity>,

// Belongs-To
#[sea_orm(belongs_to, from = "user_id", to = "id")]
pub user: HasOne<super::user::Entity>,

// Many-to-Many
#[sea_orm(has_many, via = "post_tag")]
pub tags: HasMany<super::tag::Entity>,

// Self-referential
#[sea_orm(self_ref, via = "user_follower", from = "User", to = "Follower")]
pub followers: HasMany<Entity>,
```

## API Changes

### Strongly-Typed Columns

**1.0**

```rust
Cake::find()
    .filter(Cake::Column::Name.contains("chocolate"))
```

**2.0**

```rust
Cake::find()
    .filter(Cake::COLUMN.name.contains("chocolate"))
```

The `COLUMN` constant provides compile-time type safety.

### ActiveModel Builder

**1.0**

```rust
let user = user::ActiveModel {
    name: Set("Bob".to_owned()),
    email: Set("bob@example.com".to_owned()),
    ..Default::default()
}
.insert(db)
.await?;
```

**2.0**

```rust
let user = user::ActiveModel::builder()
    .set_name("Bob")
    .set_email("bob@example.com")
    .insert(db)
    .await?;
```

### Entity Loader

**1.0**

```rust
// N+1 problem or manual joins
let users = User::find().all(db).await?;
for user in users {
    let posts = Post::find()
        .filter(Post::Column::UserId.eq(user.id))
        .all(db)
        .await?;
}
```

**2.0**

```rust
// Automatic optimization
let users = user::Entity::load()
    .with(post::Entity)
    .all(db)
    .await?;
```

### DeriveValueType

**1.0**

```rust
#[derive(Clone, Debug, PartialEq, DeriveValueType)]
pub struct Speed(Decimal);

// Manual impls needed
impl NotU8 for Speed {}
impl IntoActiveValue for Speed { /* ... */ }
impl TryFromU64 for Speed { /* ... */ }
```

**2.0**

```rust
#[derive(Clone, Debug, PartialEq, DeriveValueType)]
pub struct Speed(Decimal);

// All impls generated automatically!
// - NotU8
// - IntoActiveValue
// - TryFromU64
```

## Feature Changes

### PostgreSQL Auto-Increment

**1.0**

- Uses `serial` / `bigserial` types

**2.0**

- Uses `GENERATED BY DEFAULT AS IDENTITY`
- For legacy behavior, enable `postgres-use-serial-pk` feature

### SQLite Integer Mapping

**1.0**

- `Integer` → `INTEGER`
- `BigInteger` → `BIGINT` (not native to SQLite)

**2.0**

- Both `Integer` and `BigInteger` → `INTEGER`
- Entity generator produces `i64` by default
- Override with `--big-integer-type=i32` flag

### Removed Features

1. **`.into_condition()`** - Use `.into()` instead
2. **`on_empty_do_nothing()`** - Use `insert_many([])` which returns `None` safely
3. **Manual trait impls for DeriveValueType** - Now auto-generated

## Migration Steps

### 1. Update Dependencies

```toml
# Cargo.toml
[dependencies]
sea-orm = { version = "2.0", features = [...] }
```

### 2. Add `#[sea_orm::model]` to Entities

```rust
#[sea_orm::model]
#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel)]
#[sea_orm(table_name = "user")]
pub struct Model {
    // ...
}
```

### 3. Move Relations to Model

Remove `Relation` enum and `Related` impls, add relation fields to `Model`:

```rust
#[sea_orm(has_many)]
pub posts: HasMany<super::post::Entity>,
```

### 4. Update Column References

```rust
// Before
.filter(user::Column::Name.eq("Bob"))

// After
.filter(user::COLUMN.name.eq("Bob"))
```

### 5. Remove Manual Trait Impl for Custom Types

If you manually implemented `NotU8`, `IntoActiveValue`, or `TryFromU64` for custom types, remove them.

### 6. Use New APIs

- Replace `.into_condition()` with `.into()`
- Replace `db.execute(...)` with `db.execute_raw(...)`
- Use `Entity::load()` for eager loading
- Use ActiveModel builder pattern

## Compatibility Notes

- Old entity format still works (deprecated but not removed)
- Old API patterns still compile but may warn
- Generated entities from `sea-orm-cli` use new format by default
